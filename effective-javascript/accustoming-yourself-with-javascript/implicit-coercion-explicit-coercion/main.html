What Are Implicit Coercion and Explicit Coercion

Usually Scripting Languages do not care much about type of a variable nor care they about operation between variables.

Most of time The Interpreter (= Engine) tries to convert one type to another if it is possible. For example between Strings and Numbers. 


+"4";              // 4 Explicit Coercion
parseInt(4);       // 4 functional way

+"four";           // NaN Explicit Coercion
parseInt("four");  // NaN functional way

4 + 4;             // 8 addition

"4" + "4";         // "44" concatenation

+"4" + +"4";       // 8 Coercion then addition

"four" + "four";   // "fourfour" concatenation

+"four" + +"four"; // NaN Coercion then addition
NaN + NaN;         // NaN  addition

Challenge


const money = "I have 4 and 4 dollars";
// how much money do we have?

Solution 1


money.split(" ")
     // since NaN is evaluated to false
     // +item makes each item to NaN if it is a string
     // thus [filter] will return only string-number
	 .filter( ( item ) => { return +item } )
     // initializing: sum = 0
     // sum = sum + ( +item )
     .reduce( ( sum, item ) => { return sum + +item }, 0 )
     // 8

Solution 2


// just match the numbers :)
money.match(/\d+/)
     // sum = 0
	 // sum = sum + (+item)
	 .reduce((sum,item) => {return sum + +item}, 0)
	 // 8

Solution 3


// just match the number :)
money.split(" ")
 	 // sum = 0
	 // sum = sum + +item
	 .reduce( ( sum, item ) => {
	 // when item is NaN then
	 // sum = sum + 0
	 return sum + ( +item || 0 ) }
	 ,0 )
	 // 8
