Differences Between Primitive and Object Wrapper

Whenever is possible we should use primitive types because they are:

  • simple to use
  • lightweight

Primitive Types Are:

  1. boolean
  2. string
  3. number
  4. null
  5. undefined

const js  = "js";                      // primitive type
const njs = new String( "js" );        // object wrapper

typeof js  === "string";               // true
typeof njs === "string";               // false

const es5 = new String( "JS" );
const es6 = new String( "JS" );
es5 === es6;                           // false, two different Object

// Pitfalls
typeof "js";                           // "string"
typeof String("js");                   // "string"
typeof String(2019);                   // "string"
typeof new String("js"); // "object"

typeof 4;                              // "number"
typeof Number(4);                      // "number"
typeof Number("js");                   // "number" NOTE!!!
typeof new Number(4);                  // "object"

typeof {} === typeof "js";             // false
typeof {} === typeof String("js");     // false
typeof {} === typeof new String("js"); // true

typeof {} === typeof 4;                // false
typeof {} === typeof Number(4);        // false
typeof {} === typeof new Number(4);    // true

Some pitfalls when working with object wrappers


const n = new Number(4);
console.log( n );           // Number { 4 }
n.valueOf();                // 4
n + 0                       // 4 === n.valueOf() + 0

const s = new String("js");
console.log( s );           // String { "js" }
s.valueOf();                // "js"
s + " es6"                  // "js es6" === s.valueOf() + " es6"

Challenge


// These two objects are equal or not?
( {} === {} );

// How about these two:
( { valueOf: () => 4 } === { valueOf: () => 4 } );

Answer

  1. false, two different objects
  2. false, two different objects, even with the same value
