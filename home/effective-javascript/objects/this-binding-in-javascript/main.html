this-binding-in-javascript

this binding in JavaScript is both interesting and meanwhile confusing. Here I will try to show you five patterns I know about this binding in JS.

  • implicit this binding
  • forward this binding
  • manual this binding
  • explicit this binding
  • super class / parent this binding

functions are object

Always remember in JavaScript that functions are a special kind of object.

function f1(){
  console.log( "f1() called" );
  f1.variable = "this is f1";
}

function f2(){
  console.log( "f2() called" );
  f2.variable = "this is f2";
}

f1();
f2();

console.log( f1.variable );
console.log( f2.variable );

// output
"f1() called"
"f2() called"
"this is f1"
"this is f2"

implicit this binding

Item 37, page 99

Every function in JavaScript has an implicit binding of this, whose value is determined when the function is called.

and also

By contrast, this is implicitly bound by the nersted enclosed function.

this.go = "this is global object";

function f(){
  this.ff = "function f()";
  ff2 = "function f() 2";
}

console.log( this.go );     // this is global object
console.log( this.ff );     // undefined 
f();
console.log( this.ff );     // function f()
console.log( this.ff2 );    // function() 2

Because we did not declare variable ff2 with any of:

  • var
  • let
  • const

it became global and was bound to global object.

We can prevent this implicit this biding inside f() by using "use string"// there.

function f(){
  "use strict";
  this.ff = "function f()";
  ff2 = "function f() 2";
}

f();        // Error: this is undefined
