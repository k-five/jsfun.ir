minimize-use-of-global-variable

While using global variable seem easy and we feel it is good idea, but in large project - large codebase - we run into many headaches.

Using global variables and modifying them in different places causes side effect which means we cannot be sure about the state of a variable.

// two global variables
let score = 10;
let base = 2

// a function to find rate
function rate(){
    return score * base;
};

// execute the rate();
// ours assumptions are
// 1. No one else uses our variables score and base
// 2. No one modifies these two variables

In large codebase these two assumptions are vary closed to be wrong.

We will have many function calls and these functions usually update variables.

So it is very hard to prevent them from unexpected side effects.

Solution

The solutions are:

  • using constant variables as much as we can
  • using local variables as much as we can

For above case, something like this:

// two global variables
const score = 10;
const base = 2

// passing arguments to the function
function rate( s, b ){
    return s * b;
};

// execute the rate();
rate( score, base );
// what we fixed
// 1. No one else CAN modify our variables score and base
// 2. Our function does not modify its variables

no keyword for variable declaration

In javascript if we name a variable but do not use the keywords:

  • var
  • let
  • const

then it becomes global!

Never use this.

// two global variables

function rate(){
    result = 0;     // becomes global
    result = score * base;
    return result;
};
